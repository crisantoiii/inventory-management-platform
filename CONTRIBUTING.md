# Contributing

## Overview

Thank you for your interest in contributing.

This project follows Clean Architecture and emphasizes consistency across modules.

---

## Development Workflow

1. Create a feature branch.
2. Implement the feature.
3. Verify the project builds.
4. Run existing tests (when available).
5. Submit a Pull Request.

---

## Coding Standards

Follow CODE_STYLE.md.

Maintain consistency with existing modules.

---

## Module Checklist

When adding a new business module:

✓ Domain Entity
✓ Repository Interface
✓ Repository Implementation
✓ Feature Handlers
✓ Razor Pages
✓ Search
✓ Sorting
✓ Paging
✓ Filtering
✓ Activate/Deactivate
✓ Documentation updates

---

## Pull Request Checklist

- Builds successfully
- No compiler warnings
- No duplicate logic
- Documentation updated
- Follows project structure

---

## Commit Messages

Examples:

feat: add supplier management module

fix: correct supplier sorting

refactor: extract shared paging infrastructure

docs: update roadmap

---

## Branch Naming

feature/customer-module

bugfix/product-search

refactor/shared-filtering

docs/readme-update

---

## Reporting Issues

Describe:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Environment
